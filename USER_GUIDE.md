# AI辅助语音自动化测试框架 - 使用指南

## 📖 简介

这是一个**基于Workflow多代理编排**的智能语音测试框架，通过AI自动化完成：
- ✅ 实时监控测试过程（日志+性能）
- ✅ 多维度错误诊断（代码逻辑+资源+时序+历史）
- ✅ 自动生成测试报告（并行计算各项指标）
- ✅ 人工审核关键节点
- ✅ 打包成桌面应用，无需安装依赖

## 🚀 快速开始

### 1. 安装应用

**Windows:**
1. 下载 `AI语音测试工具-Setup-1.0.0.exe`
2. 双击运行安装程序
3. 完成安装后，桌面会出现快捷方式

**macOS:**
1. 下载 `AI语音测试工具-1.0.0.dmg`
2. 打开DMG文件
3. 将应用拖拽到Applications文件夹

### 2. 启动应用

双击桌面图标或从开始菜单启动应用。

**首次启动**：
- 应用会自动启动后端服务（约需3秒）
- 前端界面自动打开

### 3. 连接测试设备

1. 确保测试设备已通过ADB连接到电脑
2. 在"设备IP"输入框中输入设备地址（如：`10.7.187.15:5555`）
3. 点击"测试连接"验证设备是否可用

## 🎯 测试流程

### 步骤1：配置测试参数

在**测试控制面板**中设置：

1. **设备IP**: 测试设备的IP地址（格式: `ip:port`）
2. **测试场景**: 选择测试类型
   - 全双工语音交互测试
   - 唤醒率测试
   - 语音识别率测试
3. **监控时长**: 测试持续时间（秒）
4. **AI分析模式**: 
   - **规则匹配模式**（推荐）: 基于规则，速度快，无API成本
   - **Claude AI模式**: 使用Claude API，更智能但需API密钥

### 步骤2：准备测试环境

点击"开始测试"前，需确认：

✅ 测试音频文件已准备好  
✅ 音响设备已连接并调试好音量  
✅ 测试环境安静（混响 < 0.3）  
✅ 准备开始播放测试音频

### 步骤3：开始测试

1. 点击 **"开始测试"** 按钮
2. 确认准备工作后，点击"已准备好，开始测试"
3. **手动播放测试音频**（框架只监控，不自动播放）
4. 观察实时日志和性能监控面板

### 步骤4：实时监控

测试运行期间，界面会实时显示：

**📋 实时日志面板**：
- 显示所有语音事件（唤醒、识别、全双工退出等）
- AI自动标注事件类型和严重程度
- 支持搜索、过滤、导出

**📊 性能监控面板**：
- CPU使用率实时曲线
- 内存使用情况实时曲线
- 关键进程资源占用表格

### 步骤5：处理告警

如果AI检测到**重大错误**（如全双工非预期退出），会弹出告警：

1. **查看告警详情**：错误类型、发生时间、影响范围
2. **人工决策**：
   - **继续测试**: 忽略告警，继续进行
   - **诊断错误**: 暂停并启动多维度诊断
   - **终止测试**: 停止测试

### 步骤6：错误诊断（可选）

如果选择"诊断错误"，框架会启动**4个AI Agent并行分析**：

1. **代码逻辑分析Agent**: 分析代码执行流程问题
2. **系统资源分析Agent**: 检查内存、CPU等资源问题
3. **时序关系分析Agent**: 分析事件时序和竞态条件
4. **历史案例分析Agent**: 对比历史问题和解决方案

诊断完成后，会显示：
- ✅ 根本原因（含置信度）
- ✅ 解决方案（按优先级排序）
- ✅ 预防措施
- ✅ 完整诊断报告

### 步骤7：生成测试报告

测试完成后，点击 **"生成报告"** 按钮。

框架会启动**4个AI Agent并行计算指标**：
1. **唤醒率分析Agent**: 计算唤醒率、失败原因等
2. **识别率分析Agent**: 计算识别准确率、常见错误等
3. **全双工稳定性Agent**: 计算稳定性、退出原因等
4. **性能分析Agent**: 分析CPU、内存趋势和瓶颈

报告生成后，会显示：
- ✅ 总体评分和评级
- ✅ 各项详细指标
- ✅ 性能分析和优化建议
- ✅ 完整Markdown报告

### 步骤8：审核报告

AI生成报告后，会**自动进行质量审核**，然后等待**人工审核**：

1. 查看报告内容
2. 检查AI质量评分
3. 决定：
   - **接受报告**: 确认报告准确
   - **要求重新生成**: 报告有问题
   - **标记需要关注的问题**: 添加批注

### 步骤9：导出报告

审核通过后，可以：
- **导出Markdown**: 适合归档和版本控制
- **导出PDF**: 适合分享和打印

- 显示CPU使用率曲线

**内存图表**：
- 显示可用内存曲线
- 检测内存泄漏趋势

**进程表格**：
- 显示关键进程资源占用
- 颜色标注异常进程

## ⚙️ 高级配置

### AI模式切换

**规则匹配模式**（默认）：
- 基于正则表达式匹配
- 速度快、成本低
- 准确率约85%
- **推荐用于日常测试**

**Claude AI模式**：
- 使用Claude API进行智能分析
- 准确率约95%
- 需要配置API密钥
- **推荐用于重要测试和深度诊断**

**配置Claude API**：
1. 打开配置文件：`config/default_config.json`
2. 设置：
   ```json
   {
     "ai_analysis": {
       "mode": "claude_api",
       "claude_api_enabled": true,
       "claude_api_key": "your-api-key-here"
     }
   }
   ```
3. 重启应用

### 测试场景自定义

可以创建自定义测试场景配置：

1. 在 `scenarios/` 目录下创建 YAML 文件
2. 参考示例：`scenarios/fullduplex_test.yaml`
3. 定义测试参数、预期事件、指标阈值等

## 🐛 常见问题

### Q1: 启动应用时后端无法启动？

**原因**: 端口被占用或防火墙阻止

**解决**:
1. 检查8000端口是否被占用：`netstat -ano | findstr 8000`
2. 关闭占用端口的进程或修改配置文件中的端口
3. 检查防火墙设置，允许应用访问网络

### Q2: 无法连接到测试设备？

**原因**: ADB连接问题

**解决**:
1. 确认设备已连接：`adb devices`
2. 如果未连接：`adb connect 10.7.187.15:5555`
3. 检查设备IP是否正确
4. 确保设备和电脑在同一网络

### Q3: 日志中文乱码？

**原因**: 编码问题

**解决**:
1. 确保设备日志编码为UTF-8
2. 重启应用

### Q4: workflow执行失败？

**原因**: Agent错误或超时

**解决**:
1. 查看控制台日志
2. 检查网络连接（Claude AI模式需要网络）
3. 减少监控时长，避免超时
4. 切换到规则匹配模式
