# 真实场景测试指南 - 音频播放测试

**版本**: 1.0  
**适用场景**: 全双工语音交互测试、唤醒率测试、识别率测试  
**测试环境**: 需要实际播放音频设备

---

## 📋 测试前准备

### 1. 硬件准备

✅ **测试设备**:
- Android智能设备（已通过ADB连接）
- 设备IP: `10.7.187.15:5555`
- 确保设备麦克风正常工作

✅ **音频播放设备**:
- 音响或扬声器（已连接到测试电脑）
- 音量调整到合适范围（建议60-70dB）
- 音响与测试设备距离：1-2米

✅ **测试环境**:
- 安静的测试环境（背景噪音 < 40dB）
- 混响 < 0.3
- 无其他人员对话干扰

### 2. 软件准备

✅ **AI语音测试工具**:
- 启动桌面应用（双击图标）
- 等待后端服务自动启动（约3秒）
- 确认设备连接状态为"已连接"

✅ **测试音频文件**:
- 格式: WAV（16kHz采样率）
- 内容: 唤醒词 + 静音1秒 + 3-5句查询语句（每句间隔10秒）
- 可选: 混淆语音（两人对话）
- 存放位置: 准备好可以快速播放

### 3. ADB连接验证

```bash
# 确认设备已连接
adb devices

# 应该看到:
# 10.7.187.15:5555    device

# 如果未连接，执行:
adb connect 10.7.187.15:5555
```

## 🎯 测试流程

### 步骤1: 启动测试框架

1. **打开应用**: 双击桌面图标 `AI语音测试工具`

2. **等待后端启动**: 
   - 观察应用窗口底部状态
   - 看到"后端服务: 运行中"即可

3. **验证设备连接**:
   - 顶部状态栏显示设备IP
   - 状态显示"已连接"

### 步骤2: 配置测试参数

在**测试控制面板**中设置：

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 设备IP | `10.7.187.15:5555` | 自动填充 |
| 测试场景 | 全双工语音交互测试 | 根据实际需求选择 |
| 监控时长 | 300秒（5分钟） | 根据音频总时长设置 |
| AI分析模式 | 规则匹配模式 | 推荐（速度快、成本低） |

### 步骤3: 准备测试音频

**标准测试音频结构**:
```
[唤醒词] → 静音1秒 → [查询1] → 静音10秒 → [查询2] → 静音10秒 → [查询3] → ...
```

**示例音频内容**:
```
"小翼小翼"（唤醒词）
[静音1秒]
"今天天气怎么样"（查询1）
[静音10秒]
"明天会下雨吗"（查询2）
[静音10秒]
"播放一首歌"（查询3）
[静音10秒]
[可选混淆语音: 两人对话]
```

**音频文件准备**:
- ✅ 音频文件已复制到桌面或易于访问的位置
- ✅ 音频播放器已打开（VLC、Windows Media Player等）
- ✅ 音量已调整好

### 步骤4: 开始测试

1. **点击"开始测试"按钮**
   - 弹出确认对话框

2. **确认准备事项**:
   ```
   请确认以下准备工作已完成：
   ✅ 测试音频文件已准备好
   ✅ 音响设备已连接并调试好音量
   ✅ 测试环境安静（混响 < 0.3）
   ✅ 准备开始播放测试音频
   ```

3. **点击"已准备好，开始测试"**
   - 框架开始监控设备日志和性能
   - 实时日志面板开始显示内容

4. **立即播放测试音频** ⭐ **关键步骤**
   - 看到日志面板有内容后，立即点击音频播放器的"播放"按钮
   - 保持安静，不要干扰测试

### 步骤5: 实时监控

测试运行期间，观察界面：

**📋 实时日志面板**:
- 🟦 蓝色事件：唤醒、识别（正常）
- 🟧 橙色事件：性能警告
- 🟥 红色事件：错误、非预期退出（需要关注）

**📊 性能监控面板**:
- CPU使用率曲线
- 内存使用曲线
- 关键进程资源占用表格

**统计信息**:
- 总事件数（实时更新）
- 关键事件数
- 性能采样数
- 运行时长

### 步骤6: 处理告警（如有）

如果测试过程中弹出**告警对话框**：

**选项1: 继续测试**
- 忽略告警，继续运行测试
- 适用于：非关键问题、可接受的警告

**选项2: 诊断错误**
- 暂停测试，启动错误诊断workflow
- 4个AI Agent并行分析错误
- 查看诊断结果后决定是否继续

**选项3: 终止测试**
- 停止测试，保存当前数据
- 适用于：发现严重问题需要修复

### 步骤7: 测试完成

监控时长结束后：

1. **自动停止监控**
   - 状态变为"就绪"

2. **查看统计数据**:
   - 总事件数
   - 关键事件数
   - 性能采样数

3. **点击"生成报告"按钮**
   - 启动报告生成workflow
   - 4个AI Agent并行计算指标
   - 约2-3分钟完成

### 步骤8: 审核报告

报告生成后弹出**测试报告对话框**：

**查看内容**:
- ✅ 总体评分（5星评级）
- ✅ 唤醒率指标
- ✅ 识别率指标
- ✅ 全双工稳定性
- ✅ 性能分析
- ✅ 完整Markdown报告

**人工审核**:
- 检查指标是否合理
- 查看AI质量评分
- 标记需要关注的问题
- 添加审核意见

**导出报告**:
- 点击"导出Markdown"保存报告文件
- 文件名：`test-report-<时间戳>.md`

## 🔍 常见测试场景

### 场景1: 全双工语音交互测试

**目标**: 测试连续多轮对话的稳定性

**音频准备**: 
- 1个唤醒词 + 5-10句连续查询
- 每句查询间隔10秒

**关注指标**:
- 全双工稳定性 > 95%
- 非预期退出次数 = 0
- 识别率 > 90%

**成功标准**:
- ✅ 全部查询都能正确识别
- ✅ 全双工模式不退出
- ✅ 无系统错误

### 场景2: 唤醒率测试

**目标**: 测试唤醒词的识别准确性

**音频准备**:
- 10-20次唤醒词（间隔10秒）
- 包含不同音量、语速、音色

**关注指标**:
- 唤醒率 > 95%
- 唤醒延迟 < 500ms

**成功标准**:
- ✅ 唤醒成功率 ≥ 95%
- ✅ 无误唤醒
- ✅ 响应及时

### 场景3: 识别率测试

**目标**: 测试语音识别的准确性

**音频准备**:
- 1个唤醒词 + 20-50句不同查询
- 涵盖各种指令类型

**关注指标**:
- 识别率 > 90%
- 常见错误类型

**成功标准**:
- ✅ 识别准确率 ≥ 90%
- ✅ 错误可接受（同音字等）

### 场景4: 压力测试

**目标**: 测试长时间运行的稳定性

**音频准备**:
- 长时间连续播放（30分钟+）
- 包含唤醒、识别、混淆语音

**关注指标**:
- 内存是否泄漏
- CPU是否过载
- 进程是否崩溃

**成功标准**:
- ✅ 无内存泄漏
- ✅ 无进程崩溃
- ✅ 性能稳定

## 🐛 常见问题处理

### 问题1: 无法连接设备

**现象**: 状态栏显示"设备未连接"

**解决步骤**:
1. 检查设备IP是否正确
2. 执行：`adb connect 10.7.187.15:5555`
3. 确认设备和电脑在同一网络
4. 重启应用

### 问题2: 日志面板无内容

**现象**: 开始测试后，日志面板一直空白

**可能原因**:
- 设备实际未连接
- 设备上没有日志产生
- 音频尚未播放

**解决步骤**:
1. 确认设备连接状态
2. 播放测试音频
3. 等待几秒后观察

### 问题3: 识别不到唤醒

**现象**: 播放唤醒词后无反应

**可能原因**:
- 音量太小
- 环境噪音大
- 设备麦克风问题
- 唤醒功能未启用

**解决步骤**:
1. 调大音响音量
2. 检查设备麦克风
3. 重新播放唤醒词
4. 查看日志中是否有唤醒尝试记录

### 问题4: 全双工频繁退出

**现象**: 每次查询后都退出全双工模式

**可能原因**:
- 查询间隔太长（超时）
- VAD（语音活动检测）配置问题
- 系统问题

**解决步骤**:
1. 缩短查询间隔（改为5秒）
2. 查看错误诊断结果
3. 检查系统日志

### 问题5: 报告生成失败

**现象**: 点击"生成报告"后出错

**可能原因**:
- 数据不完整
- Workflow执行失败

**解决步骤**:
1. 检查是否有测试事件数据
2. 查看控制台错误信息
3. 重试生成报告
4. 联系技术支持

## 🎯 测试技巧

### 技巧1: 分段测试

**不要一次性播放很长的音频**，建议：
- 先测试唤醒（1-2分钟）
- 再测试单轮对话（2-3分钟）
- 最后测试多轮对话（5-10分钟）

**好处**: 容易定位问题、节省时间

### 技巧2: 对比测试

**准备多个版本的测试音频**:
- 标准发音版本
- 快速发音版本
- 慢速发音版本
- 不同音色版本

**好处**: 全面评估系统鲁棒性

### 技巧3: 记录环境

**每次测试记录环境参数**:
- 环境噪音级别
- 音响音量
- 设备距离
- 其他干扰因素

**好处**: 便于问题复现和分析

### 技巧4: 保存原始数据

**导出测试数据**:
- 导出日志文件（JSON格式）
- 导出测试报告（Markdown格式）
- 截图关键界面

**好处**: 便于后续分析和归档
